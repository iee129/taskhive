# 사용자 스토리

## 인증

**US-AUTH-01**
> As a **신규 사용자**,  
> I want to **이메일과 비밀번호로 회원가입**,  
> So that **서비스를 이용할 수 있다**.

수락 기준:
- 이름, 이메일, 비밀번호(8자 이상) 입력 후 가입 가능
- 이미 존재하는 이메일로 가입 시 400 오류 반환
- 가입 성공 시 JWT 토큰 즉시 반환

---

**US-AUTH-02**
> As a **기존 사용자**,  
> I want to **이메일과 비밀번호로 로그인**,  
> So that **내 데이터에 접근할 수 있다**.

수락 기준:
- 올바른 자격증명으로 로그인 시 JWT 반환
- 잘못된 비밀번호 입력 시 401 오류 반환
- 반환된 JWT로 보호 API 접근 가능

---

## 태스크 관리

**US-TASK-01**
> As a **로그인한 사용자**,  
> I want to **새 태스크를 생성**,  
> So that **해야 할 일을 기록할 수 있다**.

수락 기준:
- 제목(필수), 설명(선택), 마감일(선택) 입력 가능
- 생성된 태스크의 초기 상태는 `TODO`
- 생성 성공 시 태스크 ID와 전체 정보 반환

---

**US-TASK-02**
> As a **로그인한 사용자**,  
> I want to **태스크 상태를 변경**,  
> So that **진행 상황을 추적할 수 있다**.

수락 기준:
- `TODO → IN_PROGRESS → DONE` 순서로 상태 변경 가능
- PUT `/api/tasks/{id}` 에 `status` 필드 전달로 변경
- 변경된 상태가 응답에 즉시 반영

---

**US-TASK-03**
> As a **팀 리더**,  
> I want to **태스크에 팀원을 담당자로 지정**,  
> So that **책임자가 명확해진다**.

수락 기준:
- `assigneeId` 필드에 사용자 ID 지정 가능
- 존재하지 않는 사용자 ID 지정 시 400 오류

---

**US-TASK-04**
> As a **로그인한 사용자**,  
> I want to **불필요한 태스크를 삭제**,  
> So that **목록을 깔끔하게 유지할 수 있다**.

수락 기준:
- `DELETE /api/tasks/{id}` 호출로 삭제
- 존재하지 않는 ID 삭제 시 404 오류
- 성공 시 204 No Content 반환

---

**US-TASK-05**
> As a **로그인한 사용자**,  
> I want to **전체 태스크 목록을 조회**,  
> So that **현재 해야 할 일을 파악할 수 있다**.

수락 기준:
- `GET /api/tasks` 호출 시 전체 태스크 배열 반환
- 각 태스크에 id, title, status, dueDate 포함
